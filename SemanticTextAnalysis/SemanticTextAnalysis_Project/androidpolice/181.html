easyHome Turns Your Samsung Device's Physical Home Button Into A Capacitive One

Ryan Whitwam
Follow View All Posts

Strange as it sounds, using the physical home button on Samsung phones can be a lot of work. Tapping a capacitive or on-screen button is faster and doesn't break up the experience as much. Well, no more. There's a new app for most 2014 and 2015 Samsung devices that turns the physical button into a capacitive one, and it works surprisingly well.

The easyHome app appears to hijack the fingerprint reader functionality to act like a button press. Simply tap the home button (don't press it), and easyHome triggers a home button press. It's faster, silent, and your button won't wear out. With the pro version ($1.69), you can also wake the screen by touching the home button (as long as you're not using fingerprint unlock) or lock it with a long-tap.

You'll need to set easyHome as an administrator app, but it only takes a moment. If you've got a compatible device (see the Play Store for a full list), you should give the free version a shot.

        Well, considering the way that fingerprint scanning works, this could never happen. First off, the fingerprint is not stored IN AND OF ITSELF. It is a mathematical representation of your print and that gets stored in a one-way in TEE (Trusted Execution Environment). Secondly, when an authentication is needed, all that gets returned from the TEE is a boolean true or false as to does the scanned print match what is stored. It is physically impossible to retrieve a print from the TEE.

        • DrM

          Interesting, really need to read more about that.

          • CoreRooted

            Because the ONLY way to access the fingerprint scanner (without root, AFAIK) is through the Samsung Pass SDK. In order to be able to scan a print, you have to go through the Enroll method (which just calls Samsung's Enroll app). The way that Samsung (and now Marshmallow) allow access to the scanner driver prevents man-in-the-middle attacks. You HAVE to use the Pass SDK (or native Marshmallow APIs if your device is on 6.0) to access the scanner.

            Even then, to try to spoof the TEE boolean request for verification of a print would absolutely take root access and A LOT of modifications to the system files to load your own API to act as a middle man between apps that request fingerprint verification (again, using the Pass SDK) and the TEE. This is damn near impossible to do without full, unlocked access to the device.
